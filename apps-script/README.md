# TG 监控看板 — Google Apps Script

给每个部门主管的 **实时监控看板**。

基于部门自己的监控 Google Sheet,**不修改任何数据**,只读取聚合。跟设置精灵同款 TG 黑蓝配色,30 秒自动刷新。

---

## 展示内容

**5 个 KPI 卡**(今日)
- 未回复预警数
- 关键词命中数
- 消息被删数
- 活跃对话数
- 平均回复时长(分钟)

**6 张图表/列表**
- 近 7 天趋势折线(未回复 / 关键词 / 删除)
- 关键词命中 Top 10
- 24 小时消息热力图
- 商务人员预警排行
- 慢响应对话 Top 10
- 最新 10 条告警

## 部署步骤(每个部门做一次,5 分钟)

### 1. 打开部门对应的监控 Google Sheet

就是 TG Monitor 写入的那份,有 `信息未回复预警悦达`、`关键词监听悦达` 等分页的那个。

### 2. 进 Apps Script 编辑器

菜单:**扩展功能 → Apps Script**

### 3. 粘贴代码

Apps Script 编辑器里会看到一个默认的 `Code.gs`。依次做:

| 本仓库文件 | 粘到哪 |
|---|---|
| `apps-script/Code.gs` | 覆盖编辑器里的 `Code.gs` |
| `apps-script/Dashboard.html` | 点左上 ➕ → HTML 文件,命名 **Dashboard**(不要 .html 后缀),粘入 |
| `apps-script/appsscript.json` | 齿轮图标(项目设置) → 勾选「在编辑器中显示 `appsscript.json` 清单文件」,然后覆盖 |

💾 全部 Ctrl+S 保存。

### 4. 授权

回 `Code.gs`,选 `getDashboardData` 函数 → 点 ▶️ 运行一次。
第一次会弹 Google 授权,一路 **允许**(包括需要点「高级 → 前往xxx(不安全)」那步,因为是你自己的脚本)。

### 5. 部署为 Web App

右上 **部署 → 新建部署 → 齿轮 → 网络应用**

- 类型:**网络应用**
- 说明:随便(例如"v1")
- 执行身份:**我**
- 谁可访问:**任何拥有链接的人**(或"任何拥有链接的 Google 用户"更安全)
- 点 **部署**

复制出来的 `Web App URL`,发给部门主管即可。

### 6. (可选)更新代码

代码改过要让旧 URL 生效新版:**部署 → 管理部署 → 铅笔图标 → 版本选「新版本」→ 部署**。URL 不变。

## 设计要点

### 零配置

- **部门后缀自动识别** — 从分页标题如 `信息未回复预警悦达` 里抠出 `悦达`
- 只要 TG Monitor 原来建的分页没改名,任何部门的 Sheet 粘贴即用

### 读端聚合,不动原数据

- 代码只 `getValues()` 读数据,不写任何一个单元格
- TG Monitor 随便继续往 Sheet 写,看板永远拿得到最新

### 性能

- 每次刷新把全 Sheet 读一次,一般部门几百到几千行,1-2 秒
- 前端每 30 秒刷新一次;按钮要手动加可后补
- 如果某部门超过 10000 行,考虑后端加 `CacheService` 缓存 30s

### 安全

- Web App 的「执行身份:我」意味着主管看看板用的是**你**的权限
- Sheet 本身只有授权账号能改;看板 URL 泄露了只能看,不能改

## 排错

### 看板打不开 / 显示空白
- 检查是否已运行过一次 `getDashboardData` 授权
- 查看 Apps Script 编辑器左侧 **执行** 标签,看最近运行日志

### KPI 全是 0
- 部门后缀识别不对 → 运行 `getDashboardData`,在 console 看 `company` 字段
- 分页名跟代码对不上 → 回 Sheet 确认有 `信息未回复预警{XX}` 这 3 个分页

### 回复时长不准 / 为 0
- 对话 tab 的方向列(B 列)必须是 `A`/`B` 开头或包含 "发"/"收"
- 检查 TG Monitor 写入格式有没有改

### 自动刷新太频繁卡顿
- Dashboard.html 末尾 `setInterval(refresh, 30000)`,改成 60000(60 秒)
