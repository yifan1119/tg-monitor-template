## 集成测试 — 4 个新 endpoint

### 1. `/api/v1/violations`

```bash
TOKEN=$(grep METRICS_TOKEN .env | cut -d= -f2)
BASE=http://localhost:5050

# A. 默认(今天)
curl -s -H "Authorization: Bearer $TOKEN" "$BASE/api/v1/violations" | jq '{ok, count}'
# 期望:ok=true, count 是今天 violation_logged 条数

# B. 自定义日期范围
curl -s -H "Authorization: Bearer $TOKEN" "$BASE/api/v1/violations?from=2026-04-01&to=2026-04-29" | jq '.count'

# C. 类型过滤
curl -s -H "Authorization: Bearer $TOKEN" "$BASE/api/v1/violations?type=no_reply" | jq '[.items[] | .type] | unique'
# 期望:["no_reply"](所有 item type 都是 no_reply)

# D. 字段完整性(随便挑一条)
curl -s -H "Authorization: Bearer $TOKEN" "$BASE/api/v1/violations" | jq '.items[0] | keys'
# 期望含:account_company / business_tg_id / owner_tg_id / created_at /
#         message_text / peer_name / stage / status (== 'violation_logged')

# E. 错日期格式 → 走默认值,不报错
curl -s -H "Authorization: Bearer $TOKEN" "$BASE/api/v1/violations?from=invalid-date" | jq '.ok'
# 期望:true(invalid 被忽略,默认今天)

# F. SQL 注入尝试
curl -s -H "Authorization: Bearer $TOKEN" "$BASE/api/v1/violations?type=' OR 1=1--" | jq '.count'
# 期望:0(type 白名单不过)
```

### 2. `/api/v1/alerts`

```bash
# A. 多条件组合
curl -s -H "Authorization: Bearer $TOKEN" \
  "$BASE/api/v1/alerts?status=violation_logged&stage=2&type=no_reply&limit=10" \
  | jq '{ok, returned, limit, offset}'

# B. limit 越界 clamp
curl -s -H "Authorization: Bearer $TOKEN" "$BASE/api/v1/alerts?limit=99999" | jq '.limit'
# 期望:1000(硬上限 clamp)

# C. limit 负数 clamp
curl -s -H "Authorization: Bearer $TOKEN" "$BASE/api/v1/alerts?limit=-1" | jq '.limit'
# 期望:1(下限 clamp)

# D. 非法 status 被忽略
curl -s -H "Authorization: Bearer $TOKEN" "$BASE/api/v1/alerts?status=DROP_TABLE" | jq '.ok'
# 期望:true(白名单不过,不返该条件,不抛错)

# E. offset 分页
curl -s -H "Authorization: Bearer $TOKEN" "$BASE/api/v1/alerts?limit=5&offset=0" | jq '.items | length'
# 期望:5
curl -s -H "Authorization: Bearer $TOKEN" "$BASE/api/v1/alerts?limit=5&offset=5" | jq '.items[0].id'
# 期望:跟 offset=0 第 6 条 id 一致
```

### 3. `/api/v1/peers`

```bash
# A. 全部
curl -s -H "Authorization: Bearer $TOKEN" "$BASE/api/v1/peers" | jq '{ok, returned}'

# B. 按账号过滤
curl -s -H "Authorization: Bearer $TOKEN" "$BASE/api/v1/peers?account_id=1" \
  | jq '[.items[] | .account_id] | unique'
# 期望:[1]

# C. 按分组过滤
curl -s -H "Authorization: Bearer $TOKEN" "$BASE/api/v1/peers?group=0" \
  | jq '[.items[] | .col_group] | unique'
# 期望:[0]

# D. 非法 account_id(不是 int)被忽略
curl -s -H "Authorization: Bearer $TOKEN" "$BASE/api/v1/peers?account_id=abc" | jq '.ok'
# 期望:true(忽略非法,返全部)

# E. 字段完整性
curl -s -H "Authorization: Bearer $TOKEN" "$BASE/api/v1/peers?limit=1" | jq '.items[0] | keys'
# 期望含:id / tg_id / account_id / account_name / name / username / col_group
```

### 4. `/api/v1/messages`

```bash
# A. 缺必填 → 400
curl -s -w "\n%{http_code}" -H "Authorization: Bearer $TOKEN" "$BASE/api/v1/messages"
# 期望最后行:400

curl -s -w "\n%{http_code}" -H "Authorization: Bearer $TOKEN" "$BASE/api/v1/messages?account_id=1"
# 期望最后行:400(缺 peer_id)

# B. 正常查询
curl -s -H "Authorization: Bearer $TOKEN" \
  "$BASE/api/v1/messages?account_id=1&peer_id=1&limit=10" | jq '{ok, returned}'

# C. deleted_only=true
curl -s -H "Authorization: Bearer $TOKEN" \
  "$BASE/api/v1/messages?account_id=1&peer_id=1&deleted_only=true" \
  | jq '[.items[] | .deleted] | unique'
# 期望:[true]

# D. 非法 account_id 转 int 失败 → dashboard_api 返 error
curl -s -H "Authorization: Bearer $TOKEN" \
  "$BASE/api/v1/messages?account_id=abc&peer_id=1" | jq '.error // "ok"'
# 期望含 "必填"

# E. 字段完整性 — 含 v3.0.9 新字段
curl -s -H "Authorization: Bearer $TOKEN" \
  "$BASE/api/v1/messages?account_id=1&peer_id=1&limit=1" \
  | jq '.items[0] | keys'
# 期望含:archive_msg_id / delete_mark_pending / deleted_at / direction /
#         media_seq / media_type / msg_id / sheet_row / sheet_written / text /
#         timestamp
```

### 5. 鉴权 / 认证

```bash
# A. 没 token → 401
curl -s -w "\n%{http_code}" "$BASE/api/v1/violations"
# 期望最后行:401

# B. 错 token → 401
curl -s -w "\n%{http_code}" -H "Authorization: Bearer wrongtoken" "$BASE/api/v1/violations"
# 期望最后行:401

# C. ?token= query string 也支持
curl -s "$BASE/api/v1/violations?token=$TOKEN" | jq '.ok'
# 期望:true

# D. 4 个新 endpoint 都进 metrics access_log
# 跑完上面所有测试后:
# 后台登入 → 设置页 → metrics 调用记录 → 应能看到 4 个 endpoint 各自的命中
```

### 6. 性能 / 边界

```bash
# A. 1000 条 alerts 拉取速度
time curl -s -H "Authorization: Bearer $TOKEN" \
  "$BASE/api/v1/alerts?limit=1000" > /dev/null
# 期望:< 2 秒

# B. peers 全表(模拟 1500 条)
time curl -s -H "Authorization: Bearer $TOKEN" "$BASE/api/v1/peers" > /dev/null
# 期望:< 1 秒

# C. messages 1000 条 + 长 text
time curl -s -H "Authorization: Bearer $TOKEN" \
  "$BASE/api/v1/messages?account_id=1&peer_id=1&limit=1000" > /dev/null
# 期望:< 3 秒,JSON 大小 < 5MB

# D. 并发 10 个请求(确认不锁 DB)
for i in $(seq 1 10); do
  curl -s -H "Authorization: Bearer $TOKEN" \
    "$BASE/api/v1/violations" > /dev/null &
done
wait
# 期望:全部 200,主业务 listener / sheets flush 不受影响
```
