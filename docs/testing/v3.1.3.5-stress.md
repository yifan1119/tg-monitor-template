## 必过场景

### 1. P0-1:sync_headers 不再清空 DB(关键场景)

**场景 1a — Sheet 空 + DB 有值,sync_headers 跑后 DB 保留**
1. dev VPS 选一个号,DB 里 `operator='测试商务'` `inspector_tg_id='@admin'`(或者 web 后台填进去)
2. Google Sheet 对应 worksheet B2/B3 设为空
3. 等 600s + 触发 sync_headers(或重启 tg-monitor 立刻跑)
4. 验证 DB.operator 仍 = `'测试商务'`(老版本会变成空)
5. **预期**:log 不出现「从 Sheets 同步: 操作人员=, 所属公司=」(空字段不写)

**场景 1b — Sheet 有值 + DB 不一致,Sheet 仍能覆盖 DB(老功能保留)**
1. DB.operator = `'A'`,Sheet B2 = `'B'`
2. 触发 sync_headers
3. 验证 DB.operator 变 `'B'`,log 出现「从 Sheets 同步: 操作人员=B, ...」

**场景 1c — Sheet 跟 DB 一致,无 UPDATE**
1. DB.operator = `'C'`,Sheet B2 = `'C'`
2. 触发 sync_headers
3. log 不出现「从 Sheets 同步: ...」(无 UPDATE)

**场景 1d — Sheet 空 + DB 也空**
1. DB.operator = `''`,Sheet B2 = `''`
2. 触发 sync_headers
3. log 不出现「从 Sheets 同步: ...」(无 UPDATE)

### 2. P0-2:pull_history 异步化,不阻塞 patrol_loop 启动

**场景 2a — 故意触发 telegram flood wait,patrol_loop 仍能起来**

dev VPS 挑一个号反复重启 tg-monitor 触发 telegram 限速。**老版本**:flood wait 累积后启动会卡 30+ 分钟。**新版本**:

1. `docker restart tg-monitor-demo`
2. 等 60s 看 docker logs
3. **预期**:
   - log 出现 `[bg_pull_history] 开始背景补拉...`
   - log 出现 `[patrol_loop] 启动,interval=60s...` 在 30s 内
   - log 出现 `[bus_sync] 本轮共同步...` 在 60-120s 内
   - log 可能仍有 `Sleeping for 29s GetHistoryRequest flood wait`,但**不影响 patrol_loop**

**场景 2b — pull_history 单号失败不影响其他号**

1. 模拟某号 session 失效
2. **预期**:log 出现 `[bg_pull_history] phone=+85x... 失败 (跳过,继续下一个): ...`
3. 其他号 pull_history 继续 + patrol_loop 正常跑

**场景 2c — pull_history 任务异常被 done_callback 捕获**

1. 故意制造 telethon 罕见异常(比如 mock listener.pull_history raise)
2. **预期**:log 出现 `[bg_pull_history] 背景任务异常退出: ...` 含完整 traceback(rule 19)
3. patrol_loop / listener / bot 仍正常跑

### 3. P0-3:_startup_health_loop 5 分钟自检告警

**场景 3a — 正常启动,5 分钟后无告警**

1. 重启 tg-monitor,启动正常
2. 5 分钟后看 log
3. **预期**:log 出现 `[startup_health] patrol_loop 5 分钟内已正常跑过,健康`
4. **预期**:预警群**没有**告警消息

**场景 3b — patrol_loop 卡死,5 分钟后推告警**

故意制造 patrol_loop 启动失败(比如改代码 raise 在第一行):
1. 改 `_patrol_loop` 第一句加 `raise RuntimeError("test")`
2. 重启 tg-monitor
3. 等 5 分钟
4. **预期**:log 出现 `[startup_health] ⚠ tg-monitor 启动 5 分钟内 patrol_loop 未运行...`
5. **预期**:预警群收到告警消息

测完撤掉 raise 测试代码。

## 回滚

升级失败:
```bash
cd /root/tg-monitor-<dept>
bash rollback.sh   # 走 update.sh 内置回滚到 OLD_SHA
```

`rollback.sh` 把 git reset 回升级前 SHA,update.sh 也回老版本(没 v3.1.3.5 fix)。麦小麦回退到 v3.1.3.4 + 临时 workaround 即可。
