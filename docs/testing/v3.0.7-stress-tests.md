## 核心行为验证

### T1:静态自查 — 改动落到位

**步骤**:在 worktree 跑
```bash
grep -rn "register_global_writer\|get_global_writer\|_GLOBAL_WRITER" .
```

**预期**:

- `is_oauth_failure`: `oauth_helper.py` 1 处定义 + 1 处文档,`sheets.py` 2 处使用(APIError 兜底 + bare Exception 兜底),`dashboard_api.py` 1 处使用
- `reload_credentials`: `sheets.py` 1 处定义 + `flush_pending` 3 处自愈调用,`web.py`/`main.py` **0 命中**(原 P0 修复后已撤掉跨进程通知)

**通过**:全部匹配预期数量,无意外位置

## OAuth 自愈路径验证

### T4:E2E 模拟 — 撞坏 token 触发自愈

**前提**:dev 容器跑起来,正常写入 Sheets 中

**步骤**:
1. 备份 `data/google_oauth_token.json`:
   ```bash
   docker exec dev cp /app/data/google_oauth_token.json /tmp/token.bak
   ```
2. 撞坏 token(改 refresh_token 字段成无效值):
   ```bash
   docker exec dev sh -c 'sed -i "s/\"refresh_token\": \".*\"/\"refresh_token\": \"BROKEN_TOKEN\"/" /app/data/google_oauth_token.json'
   ```
3. 等 1-2 个 `SHEETS_FLUSH_INTERVAL` 周期(默认 5 秒,等 15 秒)
4. `docker logs dev | grep -E "RefreshError|oauth_reload"`

**预期**:
- 看到 `[sheets_flush] account=N OAuth 刷新失败,尝试自愈: ...invalid_grant...`
- 看到 `[oauth_reload] 凭证不存在` 或 `[oauth_reload] 重建失败 ...invalid_grant...`(因为 token 是坏的,reload 也会失败,这是预期)
- 看到 `[sheets_flush] account=N OAuth 自愈失败,等待 web 端 reauth`
- **不能**看到 `_flush_backoff_level` 涨高(自愈路径不进退避)

**通过**:OAuth 失效路径走对,自愈尝试触发,失败时优雅记日志而非崩

### T5:E2E 恢复 — web reauth 触发文件 IPC + 自愈

**前提**:T4 之后(token 仍坏的状态)

**步骤**:
1. 浏览器进 web → 设置页 → 重新点「连接 Google Drive」走完 OAuth(或直接撤销+重新授权)
   — 这一步 `oauth_helper.save_token()` 会把新 refresh_token 写到共享 docker volume `/app/data/google_oauth_token.json`(tg-monitor 容器同 volume mount)
2. **不要**手动 docker restart
3. 等 5-30 秒(默认 SHEETS_FLUSH_INTERVAL=5s × 1-2 个周期 + 队列里要有消息触发首次写)
4. `docker logs tg-monitor-<部门> | grep -E "RefreshError|oauth_reload|批量写入"`

**预期**:
- 看到 `[sheets_flush] account=N OAuth 刷新失败,尝试自愈: ...invalid_grant...`(用旧 gc 的最后一次失败)
- 看到 `[oauth_reload] gc 已重建, backoff 已清空 (累计 N 次)`(自愈成功)
- 看到 `[sheets_flush] account=N OAuth 自愈成功,下轮重试`
- 下一轮 flush:`批量写入 X 条消息到 Sheets` 出现
- 驾驶舱「待写入」数字开始持续下降

**通过**:授权完后**不需要 docker restart**,Sheets 写入 5-30 秒内自动恢复

**注意**:如果队列里完全没消息(从客户授权那刻起一直到测试结束都没新消息进来),flush_pending 不会主动 raise → 看不到自愈 log。这是设计:没消息要写就没问题。要复现 T5 必须在 reauth 之后向某个监控账号发一条 inbound 消息触发首次写

### T6:`is_oauth_failure` helper smoke test

**步骤**:
```python
import oauth_helper

# 真 OAuth 失败 → True
assert oauth_helper.is_oauth_failure("invalid_grant: Token has been expired or revoked.") is True
assert oauth_helper.is_oauth_failure("RefreshError: invalid_grant") is True
assert oauth_helper.is_oauth_failure("Token has been expired and 401") is True
assert oauth_helper.is_oauth_failure("OAuth credentials revoked by user") is True

# 假阳性测试 — 这些不该匹配 (原 v3.0.6 dashboard_api.py 的 bug 案例)
assert oauth_helper.is_oauth_failure("429 quota exceeded") is False
assert oauth_helper.is_oauth_failure("Sheet 404 not found") is False
assert oauth_helper.is_oauth_failure("") is False
assert oauth_helper.is_oauth_failure(None) is False

# 词汇表跟 sheets/dashboard 共享 (单一来源)
markers = oauth_helper.OAUTH_FAIL_MARKERS
assert "invalid_grant" in markers
assert "refresh_error" in markers
```

**预期**:全部断言通过

**通过**:helper 行为正确,无 v3.0.6 dashboard_api 的 dead-match bug

**通过**:429 路径完全不变

**前提**:能模拟 `oauth_helper.get_credentials()` 返 None

**预期**:返 False,`writer.gc` 不变

**通过**:失败兜底有效
