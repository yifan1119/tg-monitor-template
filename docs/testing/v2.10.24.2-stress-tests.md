# v2.10.24.2 压力测试清单

**发布前必过**。对应 ADR-0009(预警分页历史空白自动回填)。

## 核心行为验证

### T1:启动时 `backfill_alert_history()` 被调用一次

**前提**:容器 DB 里有账号且有对应预警分页

**步骤**:
1. 冷启动容器
2. `docker compose -p tg-<部门> logs --since 2m tg-monitor | grep "backfill_alert_history"`

**预期**:
- 日志至少出现一行 `backfill_alert_history: 完成 — 补填 N 行,跳过 M 行`
- 如果有可回填行,还会有 `「关键词监听xxx」补填 N 行` 之类

**通过**:启动日志里能看到 `backfill_alert_history: 完成`

### T2:幂等 — 跑两次不改任何东西

**步骤**:
1. 冷启动 → 等 2 分钟 → 确认第一轮跑完
2. `docker compose -p tg-<部门> restart tg-monitor` → 再等 2 分钟
3. 对比第一次 / 第二次 log 的 `补填 N 行` 数字

**预期**:第二次 `补填 0 行, 跳过 M 行`(M 跟第一次 skipped 相同)

**通过**:第二轮不会重复改已经补好的行

### T3:`BACKFILL_ALERT_HISTORY=false` 完全停掉

**步骤**:
1. `.env` 加 `BACKFILL_ALERT_HISTORY=false` → `./update.sh` 或 restart
2. 等 2 分钟

**预期**:日志不出现 `backfill_alert_history: 完成`,但启动其他流程(ensure_alert_tabs / ensure_account_tabs)照常

**通过**:开关生效,回填完全不跑

### T4:`_alert_backfill_loop` 独立 loop 每 1 小时跑一次(默认)

**步骤**:
1. 冷启动默认配置
2. 观察 70 分钟的日志

**预期**:
- 启动时 1 次(sheets.__init__ 调用)
- 启动后 10 分钟第一次独立 loop 触发(~ t=610s)
- 60 分钟后(~ t=4210s)第二次触发

**通过**:70 分钟内共 3 次 `backfill_alert_history: 完成` 调用

### T5:启动异常不阻塞

**场景**:模拟 Sheets API 挂

**步骤**:
1. 临时把 `sheets.spreadsheet.worksheets()` mock 成 `raise RuntimeError("mock")`
2. 冷启动容器

**预期**:
- 日志出现 `backfill_alert_history 启动时失败(不阻塞启动)` warning
- 但其他 loop(`_patrol_loop` / `_sheets_flush_loop` 等)正常启动
- 容器不崩

**通过**:异常被捕获,容器正常运行

## 数据正确性验证

### T6:只填空栏,有值的行不动

**前提**:手动造数据 —— 在「关键词监听<部门>」随便两行
- 行 A:A 栏空,B 栏空,C 栏有外事号(DB 里有 operator/company)
- 行 B:A 栏「悦达渠道」,B 栏「王勃」,C 栏有外事号(DB 里 operator 是「张伟」不同)

**步骤**:
1. restart 容器触发 `backfill_alert_history`
2. 对比前后

**预期**:
- 行 A:A/B 栏被 DB 值补上
- 行 B:**完全不变**(因为两栏都有值,无论跟 DB 一不一致)

**通过**:行 B 没被覆盖,客户手改值安全

### T7:DB 里也空的外事号 → 计入 skipped + log 清单

**前提**:造一行:C 栏是 `某外事号_A`,且 DB 里 `accounts` 表的 `某外事号_A` 的 `operator` / `company` 都是空

**步骤**:
1. restart
2. 查日志

**预期**:
- 该行不会被动(因为 DB 也空没法补)
- log 出现 `共 N 个外事号 DB 里 operator/company 都空,无法回填,客户需要去...: ['某外事号_A', ...]`

**通过**:跳过正确,log 给出清单

### T8:部分数据 → 只补缺的那一栏

**前提**:
- 行 X:A 栏「悦达渠道」,B 栏空,C 栏 `某外事号_B`(DB operator=「张伟」company=「悦达渠道」)

**步骤**:restart,查该行前后

**预期**:A 栏保持「悦达渠道」不动,B 栏补成「张伟」

**通过**:只动空栏

## 配额验证

### T9:三个分页回填共 ~7 次 API 调用

**步骤**:
1. 在 Google Cloud Console → Sheets API → Metrics 监控 read/write requests per minute
2. 手动触发 restart → 观察 1-2 分钟内峰值

**预期**:
- read:1 次 `worksheets()` + 3 次 `get_all_values()` = 4 次
- write:3 次 `batch_update()`(最多)= 3 次
- 总 ~7 次,分散在 2 分钟内,平均 ~3-4/min
- **无 429**

**通过**:峰值 < 60/min,无 429

### T10:`BACKFILL_ALERT_INTERVAL_SEC=600` 短间隔不打爆配额

**步骤**:
1. `.env` 加 `BACKFILL_ALERT_INTERVAL_SEC=600` restart
2. 观察 30 分钟配额消耗

**预期**:30 分钟内 5 轮回填 × 7 次 = 35 次 API,平均 ~1.2/min,加上其他 sync_headers / peer_name 总量还是 < 50/min

**通过**:短间隔下配额仍安全
