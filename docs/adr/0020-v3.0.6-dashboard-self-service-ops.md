# ADR-0020: v3.0.6 驾驶舱三件套运维自助化(日志面板 + Sheet 诊断 + REMIND_DELETE UI)

## 背景

一天之内踩到三个客户体感问题,都指向同一个根因: **运维诊断和小配置必须 SSH 登服务器**,客服和业务方不会用终端,反馈链路变成"客户找你 → 你找代码 → 让客户跑命令 → 客户反馈输出 → 你再定位"来回三四次,一个小问题花半小时。

本轮记录下的三个反馈:
1. **"怎么查 log?"** — 客户 Sheet 卡了 3 小时,想看监听器为什么没写入,但后台没有日志查看入口
2. **"Sheet 写入健康显示积压 652 条但不知道为什么"** — 驾驶舱卡片只有数字没解释,客户不知道该等 / 重新授权 / 还是联系你
3. **v3.0.5 加了 `REMIND_DELETE_TEXT` 但忘了在设置页开放输入框** — 要改文案还得 SSH 编辑 .env + 重启容器

## 决策

一次 v3.0.6 把这三个都做掉,统一主题:**驾驶舱三件套运维自助化**。

### 1. 后台日志查看面板

`dashboard.html` 底部加折叠面板(默认收起,点击展开):
- 下拉选容器(tg-monitor-<部门> / tg-web-<部门> / tg-caddy-*)
- 行数选择(100/200/500/1000/2000,默认 200)
- 关键词过滤(可选,回车即查)
- 刷新按钮 + 自动刷新 checkbox(5 秒一次)
- `<pre>` 区域展示带时间戳的原文 log

**后端**:
- `dashboard_api.container_logs(name, tail, grep)` — 读容器 log,支持过滤
- `dashboard_api.list_containers()` — 列本部门相关容器给 dropdown
- `web.py` 两个路由 `/api/diag/logs` / `/api/diag/containers`(都 `@login_required`)

**安全约束**(最重要):
```python
allowed_exact = {f"tg-monitor-{company}", f"tg-web-{company}", f"tg-caddy-{company}"}
if container_name not in allowed_exact and not re.fullmatch(r"tg-caddy-[A-Za-z0-9_\-]+", container_name):
    return {"error": "不允许查看容器 ..."}
```
- **白名单校验先于 docker 模块导入** — 越权请求永远被拦,跟 docker SDK 可用性无关
- 允许 `tg-caddy-<任意部门>` 正则是因为 shared caddy 场景下客户要看共享的 caddy log
- 其他 `tg-caddy-xxx` 之外的命名(比如客户自建的 `caddy-my-blog`、`nginx-proxy`)全部拒绝
- 正则使用 `re.fullmatch(r"tg-caddy-[A-Za-z0-9_\-]+")` 拒绝 shell 注入字符(`;` `|` `$` 等)
- `tail` 夹到 10-2000 防爆

### 2. Sheet 写入堵塞自动诊断

`dashboard_api.sheets_health()` 扩展,不再只返回计数。加入 `_diagnose_sheets_stuck()`:

**判断阈值**:
- `pending < 50` → 正常,不诊断(省开销)
- `pending >= 50` 且 `now - last_write < 15min` → 慢但在写,状态 warning 不需要动作
- `pending >= 50` 且 `age >= 15min` → 确认堵塞,扫 tg-monitor 最近 1 小时 log 找错误模式

**错误模式匹配**:
- `invalid_grant` / `token has been expired` / `refresh_error` / `credentials do not contain` / `401` → OAuth 失效,action=`reauth`
- `429` / `rate_limit` / `quota exceeded` / `user_rate_limit_exceeded` → 429 限流,action=`wait`
- `not found` / `permission denied` / `does not have access` / `404` / `403` → Sheet 权限问题,action=`check_sheet`
- 其他 → 通用 warning

**前端**:
`renderSheets()` 新增 `sh-diag` 条
- `status=ok` → 不显示
- `status=error` → 红色条 + 修复按钮(`reauth` → 去设置页;`check_sheet` → 去设置页检查 Sheet)
- `status=warning` → 黄色条 + 「自愈中」提示(wait 类)

### 3. REMIND_DELETE_TEXT UI 输入框

`templates/setup.html` 在「两段式未回复预警」section 加第三个 textarea:
```html
<div class="field">
  <label>删除消息提醒文案</label>
  <textarea name="remind_delete_text" id="remind_delete_text" ...>{{ d.remind_delete_text }}</textarea>
  <div class="hint">... ⚠ 要先配好负责人 TG ID 才会走 @负责人 格式。</div>
</div>
```

`web.py`:
- GET handlers 两个(`/setup` 和 `/settings`)都加 `"remind_delete_text": env.get("REMIND_DELETE_TEXT", "")`
- POST handler(`/api/settings/update`)加 `updates["REMIND_DELETE_TEXT"] = form.get("remind_delete_text")`

热 reload 已经 v3.0.5 加好(`config.reload_if_env_changed` 读这个 env)。

## 原因

1. **运维体验是产品的一部分** — 一个功能本身做对了但出问题时客户没自查工具,用户体验一样差。今天半天时间大部分花在"客户反馈 → SSH 查 log → 解释原因"这种低价值往返,写自助化工具值。

2. **安全边界要清晰** — 后台看 log 是敏感能力(能看到 bot token、账号手机号、客户对话)。所以用**双重白名单**:外层 `@login_required` 防未登录;内层容器名正则校验防越权。`tg-caddy-*` 用正则而不是精确匹配是因为共享 caddy 模式下客户共用一个 caddy 容器,但用正则限定前缀避免看到其他项目的 caddy。

3. **诊断只在堵塞时扫 log** — sheets_health 每 5 秒被驾驶舱 poll 一次,如果每次都扫 log 会拖慢响应。阈值(`pending >= 50` + `age >= 15min`)确保正常态完全不触发 docker API,只有真堵塞才跑完整诊断。

4. **UI 缺口补得越早越好** — v3.0.5 的 `REMIND_DELETE_TEXT` 环境变量没对应 UI 是遗漏,v3.0.6 补上才完整。不等下个大版本。

## 后果

### 正面
- 客户遇到未回复 / 删除预警 / Sheet 卡顿 / HTTPS 故障等,**第一时间自己浏览器打开驾驶舱看 log 和 Sheet 诊断**,不用找你
- Sheet 堵塞第一眼就能看到具体原因 + 修复入口,OAuth 失效场景直接点按钮重新授权
- 删除消息预警文案修改成为纯 UI 操作,彻底无需 SSH

### 潜在问题 / trade-off
- **Log 面板看到的是容器原始 log**(包含 JSON 格式的 TG 消息片段、bot token 前缀等) — 客户自己的部门数据看自己的 OK,但如果之后加团队协作要确保所有 web 账号都是被信任的人
- **诊断扫 log 会调 Docker API** — pending >= 50 才触发,理论上每个 /api/dashboard/snapshot 请求最多一次 docker.logs 调用。驾驶舱 5 秒 poll → 极端情况下 12 次/分钟的 docker API 调用,对 Docker daemon 完全无压力但值得注意
- **诊断是关键词匹配**,不是解析 Google API response structure。TG-monitor 底层库(gspread 等)版本升级改了 log 格式理论上会导致诊断漏判 → 降级为显示通用 warning 而不是报错,可接受

### 未涉及(刻意不做)
- 日志实时 streaming(WebSocket):5 秒 poll 对一般场景够用,streaming 加复杂度不划算
- Log 下载 / 导出功能:浏览器 `Ctrl+A` + `Ctrl+C` 复制足够,不单独做下载按钮
- 「今日 dedupe 状态」UI(之前提过的)— 单独 v3.0.7 做,本版已经够密

## 升级指引

```bash
cd /root/tg-monitor-<部门> && ./update.sh
```

零感知,三个功能升级后全部可用:
- 打开驾驶舱底部「日志查看」面板就能看 log
- Sheet 写入健康卡片如果堵会自动显示诊断条
- 系统设置页「两段式未回复预警」区块多了「删除消息提醒文案」输入框
