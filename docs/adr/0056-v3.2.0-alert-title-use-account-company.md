# ADR-0056 · v3.2.0 预警标题用 account.company 跟正文一致(跨公司账号修正)

## 背景

v3.1.7 ADR-0050 修了预警**正文**「中心/部门」从 `account.company` 读 + swap 显示「中心-公司」顺序。但**标题**「【信息未回复预警 XXX】」里的 XXX 一直用 `config.COMPANY_DISPLAY`(dept .env 配的整体标签)。

2026-05-19 客户反馈预警截图:
- 标题:「【信息未回复预警**渠道中心-恒睿公司**】」(dept 名)
- 正文:「中心/部门:**伊甸维度-渠道中心**」(账号归属)
- 不一致

根因:某个外事号物理跑在恒睿公司渠道中心 dept 上,但**归属字段被填到伊甸维度**(跨公司账号场景)。中央台路由 / 预警群路由都按 account.company 走,但预警标题文案模板没跟上。

## 决策

`templates.py` 加 `_alert_title_label(company)` helper:

```python
def _alert_title_label(company):
    swapped = _swap_company(company)
    return swapped or getattr(config, "COMPANY_DISPLAY", "") or ""
```

6 处预警标题(`no_reply_alert` / `no_reply_alert_stage1` / `no_reply_alert_stage2` / `delete_alert` × 2 / `keyword_alert`)从 `{config.COMPANY_DISPLAY}` 改成 `{_alert_title_label(company)}`。

## 原因

1. **标题正文同源**:都用 account.company → 跨公司账号场景预警全文一致
2. **fallback 兼容**:account.company 空 → 退回 dept config.COMPANY_DISPLAY,老 dept 没配归属零行为变化
3. **复用 `_swap_company`**:跟正文同一 swap 逻辑,「公司-中心」存储 → 显示「中心-公司」
4. **不动 sendMessage / 预警群路由**:这两个由 bot.py + 中央台 alert_routes 负责,本版只动文案显示

## 来源

- 2026-05-19 客户截图反馈预警标题跟正文不一致
- v3.1.7 ADR-0050 已规范化正文,标题被遗漏

## 后果

**代码改动**(超小):
- `templates.py`:+8 行(新 helper)+ 6 处标题字符串替换

**升级路径**:
- 客户跑 `hotfix_v3.1.8.sh`(默认 TARGET_TAG=v3.2.0)→ 只 recreate tg-monitor,HTTPS 零变动
- 下一条预警立刻显示标题正文一致(跨公司账号修正)

**风险**:
- account.company 空 → fallback dept COMPANY_DISPLAY(跟升级前一样,零变化)
- 客户跨公司账号场景预警标题立刻变,可能困惑(但跟正文已经一致 = 用户期望)

**回滚**:`git reset --hard v3.1.9` + `docker compose up -d --no-deps --force-recreate tg-monitor`
