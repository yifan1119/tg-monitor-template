# ADR-0048 · v3.1.4 Service Account 替代 OAuth(Sheets 写入永不失效)

## 背景

OAuth 反复救火不可持续:
- **7 天 Testing 模式过期**:客户 GCP 项目没发布(/ 发布到 "Production" 要 Google 审核),refresh_token 自动 7 天过期
- **50 refresh_token 上限**:同账号同 client_id 累计签发超 50 个自动 revoke 老的
- **风控 revoke**:Gmail 反复授权同 OAuth client 触发 Google 安全系统,新 token 几分钟 ~ 几小时被自动 revoke

2026-04-29 ~ 2026-05-15 期间陈家碧 / 苏木 / 季霖 / 吴苍河 / 麦小麦 5 客户反复 OAuth 失效,每周救火,客户运维痛苦,中央台 collector 反复 401。

之前(v3.0.7 / v3.0.6 / v3.0.21 等)做过的局部 fix:
- v3.0.7 OAuth 重新授权后 Sheets 自愈
- v3.0.6 Sheet 写入堵塞诊断
- 但**根因没解决**(OAuth 本身机制脆),客户需要每周 SSH 重新授权

## 决策

**Sheets 写入凭据从 OAuth(用户授权 token)切到 Service Account**:

1. `sheets.py:SheetsWriter.__init__` 启动检测 `data/service_account.json`:
   - 有 SA 文件 → `google.oauth2.service_account.Credentials.from_service_account_file` → SA 模式连接,日志 `Google Sheets 连接成功 (Service Account): <sa_email>`
   - 无 SA 文件 → 老 OAuth 路径(向后兼容,老客户升级零感知)
2. **每中心一个 SA**(商务 / 渠道 / 运营 / 恒丰公司):
   - 配额隔离(60 / SA / min,Google 给):一个中心吃光配额不影响其他中心
   - 客户 dept 按 dept.center_id 拉对应中心的 SA
3. **中央台 SA 分发**(配套 tg-central-dashboard v0.20+):
   - `/api/v1/sa/<center_slug>` 端点用 `departments.metrics_token` 鉴权
   - dept 跨中心隔离(v0.24 P0-1 加,v0.38 取消改成 log 提示)
4. **客户操作 3 步**:
   - dept web 后台 `/settings` 「所属中心」选 → 点「拉取该中心 SA 配置」
   - 中央台返 SA JSON → tg-monitor 自动写 `data/service_account.json`(chmod 600 + atomic write tmp + os.replace + 老 SA 自动备份 `service_account.json.bak.<ts>`)
   - 把 SA email 加到客户 Google Sheet 协作者(编辑权)→ SA 模式立刻能写

## 原因

1. **SA 永不失效**:后端到后端 service-to-service,Google 不卡 token 过期 / refresh 上限 / 反垃圾风控
2. **配额可控**:每中心一个 SA 配额隔离,不撞顶
3. **运维零干预**:客户不再周期性重新授权,告别 OAuth 救火
4. **向后兼容**:老客户没 SA 文件 → 继续走 OAuth 老路径,不强制迁移

## 来源

- 客户反馈(2026-04-29 ~ 2026-05-15)反复 OAuth 失效
- Codex round1 review P0:URL 推导 rstrip("/") + SA JSON 缺关键字段不当「已配」+ HYPERLINK label 过滤换行
- 配套中央台 v0.20+(SA 分发 endpoint)

## 后果

**改动**:
- `sheets.py` SheetsWriter __init__ SA fallback(38 行新增)
- `tasks.py _resolve_inspector_display` 改 HYPERLINK 跳 t.me 链接 + 过滤 `\n\r\t` 防破公式
- `templates/setup.html` 加「所属中心」下拉 + 「拉取该中心 SA 配置」按钮 + SA 共享指引 + 黄色 warning(点击 SA email 即复制)
- `web.py` 加 `/api/setup/fetch-sa` 路由 + `_get_sa_status` helper(client_email + private_key 校验防 SA 损坏误判)

**风险与缓解**:
- SA JSON 含私钥 → chmod 600 + atomic write + .gitignored(`data/`)
- 拉错中心 SA → Google Sheet 写时 403 PERMISSION_DENIED 立刻报错(隔离在 Sheet 协作者层)
- 客户没加 Sheet 协作者 → log 报错引导加协作者

**遗留**:
- 全网客户都升 v3.1.4 后才能用 SA(老客户升 v3.1.3.x 仍走 OAuth)
- SA 拉取需要中央台 v0.20+ 提供 endpoint,中央台 down 时拉不到(但不影响已有 SA 文件继续生效)
