# ADR-0057 · v3.2.1 web 后台 modal 公司+中心下拉格式 fix(跟 v3.1.7 后端对齐)

## 背景

v3.1.7(ADR-0050)改 `tasks.py:_sync_account_business_to_sheet` 写 Sheet B3 时转「中心/公司」格式(`/` 分隔,中心在前),客户期望统一显示「中心/部门」。同时 `templates.py:_swap_company` 让正文显示反转后的「中心-公司」。

**但 `templates/index.html` 的 modal JS 没跟着改**:
- 加载逻辑(line 1137-1153)用老「公司-中心」格式 `endsWith` 中心匹配:对「运营中心/蝴蝶效应1」格式 endsWith('中心')=False → match 失败 → 下拉显示「-- 选公司 --」「-- 选中心 --」默认空
- 保存逻辑(line 1186-1194)用 `c1 + '-' + c2`(公司-中心)拼接 → 客户保存归属后 DB 存「公司-中心」老格式

2026-05-19 客户反馈:**在 web 后台填好归属保存后,过一会儿(下次打开)又显示空**。根因 modal JS 加载找不到 match → 显示默认空 → 客户没意识到 + 又点保存 → 前端发空字符串 → DB.company 被清空。

## 决策

`templates/index.html` JS 改 3 处:

### 1. 加载兼容新格式 + 老格式

```javascript
// v3.2.1: 先尝试「中心/公司」(startsWith)
for (const opt of centerSel.options) {
    const v = opt.value;
    const keyword = v.endsWith('中心') ? v.slice(0, -2) : v;
    if (raw.startsWith(v + '/') || raw.startsWith(keyword + '/')) {
        matchedCenter = v;
        const prefixLen = raw.startsWith(v + '/') ? v.length + 1 : keyword.length + 1;
        matchedCompany = raw.slice(prefixLen).trim();
        break;
    }
}
// 没匹配新格式 → 试老格式「公司-中心」(endsWith)
if (!matchedCenter) {
    for (const opt of centerSel.options) {
        const v = opt.value;
        const keyword = v.endsWith('中心') ? v.slice(0, -2) : v;
        if (raw === v || raw.endsWith(v) || raw.endsWith(keyword)) {
            matchedCenter = v;
            let prefix = raw.slice(0, raw.length - (raw.endsWith(v) ? v.length : keyword.length));
            prefix = prefix.replace(/[-\/\s]+$/, '').trim();
            matchedCompany = prefix;
            break;
        }
    }
}
```

### 2. 保存用「中心/公司」标准格式

```javascript
// 中心 + "/" + 公司(跟 v3.1.7 sheets.py 同步格式一致)
const combined = c1 && c2 ? (c2 + '/' + c1) : (c2 || c1 || '');
```

### 3. 提示文案改

```
💡 选公司 + 选中心 → 系统自动拼成「中心/公司」。
```

## 原因

1. **跟后端对齐**:v3.1.7 sheets.py 同步格式「中心/公司」,modal 保存用同一格式,避免 swap 不必要
2. **加载兼容两格式**:dept DB 里可能新老混用,modal 都能识别 → 客户改归属不丢失
3. **中央台 fuzzy match 兼容两格式**(v0.23),所以 DB 用任一格式都能路由

## 来源

- 2026-05-19 客户反馈 hudiexiaoying1「暴走吃瓜网-火火」归属保存后失效
- 实测 DB.company 在 modal 操作后被清空,根因前端 JS 加载逻辑跟后端不对齐

## 后果

**代码改动**:
- `templates/index.html`:~30 行(line 1130-1163 加载逻辑 + line 1186-1194 保存逻辑 + 提示文案)

**升级路径**:
- 客户跑 `hotfix_v3.1.8.sh`(默认 TARGET_TAG=v3.2.1)→ 只 recreate tg-monitor(包含 tg-web 容器读模板)
- 升级后客户打开 modal:看到正确公司+中心下拉值 → 保存不再清空

**风险**:
- 老格式 DB(「公司-中心」)依然能加载 + 客户保存时转新格式覆盖,**单向迁移**:DB 逐渐从老格式迁到新格式
- 中央台 fuzzy match 兼容两格式,过程中路由不挂

**回滚**:无 schema 改,git reset 老 templates/index.html 即可
