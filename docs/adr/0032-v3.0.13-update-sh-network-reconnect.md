# ADR-0032 · v3.0.13 · update.sh 共享 Caddy 模式 web 容器 network 重连自愈

## 背景

某客户线上反馈:跑 `./update.sh` 升级 v3.0.12 后,web 后台访问出 **502 Bad Gateway**。
SSH 上去看 Caddy 日志反复在喊:

```
"msg":"dial tcp: lookup tg-web-<部门> on 127.0.0.11:53: server misbehaving"
"status":502
```

Caddy 走 Docker 内部 DNS(`127.0.0.11`)解析容器名失败 → 反代失败。
而 `tg-web` 容器自己**正常活着**(127.0.0.1 GET /login 一直 200)。

## 根因

「**一台 VPS 多部门 + 共享 Caddy**」模式部署时:

1. 首次部署跑 `enable_https.sh` — 把 web 容器 `docker network connect` 到共享 Caddy 所在 network
2. 客户跑 `./update.sh` 升级 — `docker compose up -d` 因 image / .env 变更 **recreate** `tg-web` 容器
3. 旧 web 容器被销毁,**外部 network connect 跟着旧容器一起丢**
4. 新 web 容器只在 docker compose default network 里,跟共享 Caddy 不通
5. Caddy 容器名 DNS 解析失败 → 502

`update.sh` v3.0.13 之前已经有 Caddyfile inode 自愈(ADR-0017)和 Caddy profile 重启(v2.10.22),但**没处理 network 重连**这个场景。

## 决策

`update.sh` 在 Caddyfile inode 自愈后插入一段 **network 重连自愈**:

```bash
if [ -n "$MY_CADDY" ] && [ "$MY_CADDY" != "tg-caddy-${COMPANY_NAME}" ]; then
    # 仅 shared 外部 Caddy 模式才需要(自建 Caddy 走 default network 不会断)
    WEB_CONTAINER="tg-web-${COMPANY_NAME}"
    # ... 检测 web 跟 Caddy 共同 network
    if [ -z "$SHARED" ]; then
        # 自动把 Caddy 接入 web 的 default network
        docker network connect "$WEB_PRIMARY" "$MY_CADDY"
        docker exec "$MY_CADDY" caddy reload --config /etc/caddy/Caddyfile
    fi
fi
```

## 原因

1. **客户零感知** — 升级后自动通,不再 SSH 修
2. **只对 shared Caddy 模式生效** — `MY_CADDY != tg-caddy-${COMPANY_NAME}` 判断,自建 Caddy 不影响
3. **方向选择**:把 **Caddy 接入 web 的 default network**(而不是反过来),语义更对 —
   web 容器是 docker compose 管理的、稳定的;Caddy 是外部容器,临时加 network 不污染部门内 compose 配置
4. **幂等** — `docker network connect` 已存在会报 "already exists",过滤掉不报错
5. **失败不阻断升级** — 检测失败、connect 失败都不抛错,只 log

## 来源

某客户 2026-04-30 现场反馈(linghang-qd 部门,共享 Caddy `tg-caddy-linghangqudao`)。

## 后果

- ✅ 共享 Caddy 模式升级后 502 自愈
- ✅ 自建 Caddy 模式不受影响
- ⚠ **第一次升级到 v3.0.13 本身仍需手工修一次**(因为是 update.sh 本身的修复,
  升级到 v3.0.13 之前 web 已经断了 network)。修法:
  ```bash
  CADDY=$(docker ps --filter name=caddy --format '{{.Names}}' | head -1)
  NET=$(docker inspect tg-web-<部门> --format '{{range $k,$v := .NetworkSettings.Networks}}{{$k}}{{end}}')
  docker network connect "$NET" "$CADDY"
  docker exec "$CADDY" caddy reload --config /etc/caddy/Caddyfile
  ```
- 之后所有版本升级都自动走 update.sh 自愈,不再需要人工

## 测试

- shared Caddy + 跑 update.sh:web 容器若不在 Caddy network → 自动 connect → reload → 502 消除
- 自建 Caddy + 跑 update.sh:跳过自愈段(MY_CADDY 等于本部门容器名)
- 已经在同 network:跳过 connect 不打扰
