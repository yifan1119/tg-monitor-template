# ADR-0029 · v3.0.10 · 略过无意义未回复告警(纯规则版)

## 背景

2026-04-29 客户反馈:未回复预警群里大量「你好 / 在? / 1 / ? / 表情包」类客户消息触发未回复告警,业务员
看到也没必要立刻回(纯问候,没业务诉求)。审计员每天审 200+ 条这种 pending,
99% 是 cancel,做了无用功。

需求:这类「无业务含义」的客户消息不该触发未回复告警。

## 决策

`tg-monitor` 加纯规则过滤层 — 在 `_no_reply_loop` 触发告警前,检查最新一条客户消息:
1. **文本黑名单命中** → 略过(默认 30 个常见问候词,客户可改 `.env::SKIP_NO_REPLY_TEXTS`)
2. **长度 ≤ 阈值** → 略过(默认 1 字符,即纯标点 "?" "!" "1")
3. **纯 emoji / 表情符号** → 略过(unicode 范围检测)
4. **media_type == sticker** → 略过(TG 表情包,无业务含义)
5. **photo / video / file** → **不**略过(可能是业务图,保守保留)

任意一条命中 → 直接 continue,不进 `send_no_reply_alert`。

LLM 模型识别在 v3.1.x 再做(成本可控但本次先纯规则,跑 1-2 周看实际数据)。

## 实现细节

### config.py

新增:
- `SKIP_NO_REPLY_TEXTS`(env)— 逗号分隔黑名单词,空时用内置 30 词默认
- `SKIP_NO_REPLY_MIN_LEN`(env,默认 1)— 长度阈值
- `SKIP_NO_REPLY_PURE_EMOJI`(env,默认 true)— 是否启用纯 emoji 检测
- `is_trivial_no_reply(text) -> bool` helper:三条规则任意命中返 True

### database.py

`get_unanswered_candidates()` SELECT 多 `m.media_type AS last_media_type`,给调用方判 sticker 用。

### tasks.py

`_no_reply_loop` 拿到 candidate 后第一步先 check sticker / trivial → continue。

## 默认黑名单(30 词)

```
你好 / 您好 / 嗨 / hi / hello / Hi / Hello / HELLO
在 / 在? / 在? / 在吗 / 在嘛 / 在么 / 在不在 / 在不
1 / ? / ? / ?? / ?? / 嗯 / 嗯嗯 / 哦 / 哦哦
好 / 好的 / 好滴 / ok / OK / okay / Okay
哈 / 哈哈 / 嘿 / 嘿嘿
```

加文本长度 ≤ 1(仅含 "?" "!" "1" "嗯" 这种单字)+ 纯 emoji 检测 + sticker 检测。

## 原因

1. **零依赖外部**:不调 LLM,无 token 成本,无网络延迟
2. **完全可控**:客户改 `.env` 立刻生效(`reload_if_env_changed`,下条消息进来就用新规则)
3. **保守保留**:photo/video/file 不当 trivial,业务图不会被误杀
4. **向后兼容**:`SKIP_NO_REPLY_TEXTS=""` + `SKIP_NO_REPLY_MIN_LEN=0` + `SKIP_NO_REPLY_PURE_EMOJI=false`
   → 行为完全 = v3.0.9.1。旧客户升级行为零变化(默认开,但客户能关)

## 来源

- 2026-04-29 客户聊天截图:「未回复的消息,比如你好一个表情包或者1或者?无关业务这种就略过过不做反馈通知可以吗」

## 后果

- 客户升级 v3.0.10 后,未回复推送量预估减少 30-60%(看具体客户群对话风格)
- 审计员每天审单量降下来,不再淹没在「你好」「在?」噪音里
- 误杀风险:短业务消息(如「问个价」3 字)可能被规则误判
  - 缓解:客户在 `.env::SKIP_NO_REPLY_TEXTS` 主动加 / 删,客户自己控
  - 缓解:`SKIP_NO_REPLY_MIN_LEN=0` 关掉长度阈值,只走黑名单 + emoji
- 漏杀风险:「你好啊~」「nh」(拼音简写)规则识别不到 → 仍触发告警
  - v3.1.x LLM 兜底层补这块

## 测试

- `is_trivial_no_reply("你好")` → True
- `is_trivial_no_reply("在?")` → True
- `is_trivial_no_reply("1")` → True (长度 1)
- `is_trivial_no_reply("?")` → True
- `is_trivial_no_reply("?")` → True
- `is_trivial_no_reply("?")` → True (单字符)
- `is_trivial_no_reply("?")` → True (中文标点单字)
- `is_trivial_no_reply("我要买货")` → False
- `is_trivial_no_reply("问个价")` → False (3 字 > min_len 1,不在黑名单)
- `is_trivial_no_reply("你好 我问下价格")` → False (整体串不在黑名单,长度 > 1)
- `is_trivial_no_reply("?")` → True (纯 emoji)
- `is_trivial_no_reply("??")` → True
- `is_trivial_no_reply("?? hi")` → False (混了非 emoji 字符 "hi")
- `media_type=='sticker'` → 任意 text 都 skip
- `media_type=='photo'` + text='' → 不 skip(业务图保留)

## 升级 / 回滚

升级:`./update.sh`,无新表无新字段无 sessions 影响。

回滚:`.env` 加三行行为完全等价 v3.0.9.1:
```
SKIP_NO_REPLY_TEXTS=
SKIP_NO_REPLY_MIN_LEN=0
SKIP_NO_REPLY_PURE_EMOJI=false
```
