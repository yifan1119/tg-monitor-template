# ADR-0049 · v3.1.5 共用 Caddy 多 dept site block 治根 — conf.d/ 目录 mount + import

## 背景

同 VPS 多 dept 共用 Caddy 的 HTTPS 接入问题已踩过 5+ 次,每次局部修没治根:

- v3.0.2 (2026-04-23): Caddy bind mount inode 断裂 → 加 inode 自愈 + caddy-doctor.sh
- v3.0.3 (2026-04-23): update.sh 升级时自动 Caddy 体检 + 自愈
- v3.1.1 (2026-05-??): Caddyfile 误 commit demo VPS 子域名 → update.sh self-heal
- v3.1.3.2 (2026-05-13, ADR-0044): 共用 Caddy DNS 撞车 → 显式容器名
- 2026-05-14 麦小麦 fanout: enable_https.sh append Caddyfile 被 git reset 冲掉
- 2026-05-14 CLAUDE.md 硬规定 #12: 「append 到 git tracked 文件必须改 .gitignored + import」
- **2026-05-18 陈家碧 187.127.114.68 3 dept HTTPS 没接进主 Caddy**: 主 Caddyfile 没他们 site block,因为 enable_https.sh 当年没跑(命名混乱 + crash loop 跳过)

CLAUDE.md 硬规定 #12 写了正确做法但主仓代码一直没落地。今天再次踩到时陈家碧 9 IP,
决定治根。

## 决策

**enable_https.sh 把 site block 写 `conf.d/<COMPANY>.caddy`**,不再 append 主 Caddyfile:

1. 主仓 `Caddyfile` 末尾加 `import conf.d/*.caddy`(glob,空目录 caddy 不报错)
2. 主仓 `docker-compose.yml` caddy volumes 加 `./conf.d:/etc/caddy/conf.d:ro`(**目录 mount**,无 inode 断裂问题)
3. `.gitignore` 加 `conf.d/*.caddy`(每 dept .caddy 不进 git,update.sh git reset 不冲)
4. `conf.d/.gitkeep` 让目录被 git track
5. `enable_https.sh` 检测外部 Caddy 是否 mount 了 conf.d/:
   - 是 → 写 `${EXT_CONFD}/${COMPANY}.caddy`(v3.1.5+ 治根路径)
   - 否 → 退回老 append 主 Caddyfile + warn(老客户兼容,推升到 v3.1.5+)

## 原因

1. **目录 mount vs 文件 mount**:目录 mount 不受 inode 断裂影响(host 上新建/删除文件容器立刻看到)。
2. **每 dept 独立 .caddy 文件**:删某 dept site block 不动其他 dept,改 / 加都 atomic。
3. **.gitignored**:`update.sh` 跑 `git reset --hard` 不冲掉(治本)。
4. **glob import**:caddy `import conf.d/*.caddy` 空 glob 合法,初次部署 conf.d 空也不挂。

## 来源

- 客户陈家碧 187.127.114.68 反馈 dept HTTPS 打不开(2026-05-18)
- CLAUDE.md 硬规定 #12 + 累计 5+ 次同模式踩坑

## 后果

**未来新部署 / 客户升级到 v3.1.5**:
- docker-compose.yml mount conf.d/ → caddy 读 conf.d/*.caddy
- enable_https.sh 写独立 .caddy 文件,update.sh git reset 不冲
- HTTPS 接入「永久持久」

**老客户(v3.1.5 前)主 Caddyfile append 了 site block**:
- 升 v3.1.5 时 git reset 会冲掉那些 append
- update.sh 不做自动迁移(防误判 v 风险),需要手术挪到 conf.d/<COMPANY>.caddy
- 陈家碧 187.127.114.68 单独手术(已 append 3 dept site block 到主 Caddyfile)

**回滚**:把主仓 Caddyfile 末尾 `import conf.d/*.caddy` 删掉 + docker-compose 删 mount + 重启 caddy。`conf.d/<COMPANY>.caddy` 仍在 host 但不再加载。
