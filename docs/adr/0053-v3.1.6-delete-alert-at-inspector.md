# ADR-0053 · v3.1.6 删除消息预警 @ 监察员(inspector_tg_id)

## 背景

v3.0.5 设计:删除消息预警跟 stage2 未回复审批体验对齐 — 账号配了 `owner_tg_id` → @ 负责人 + 「登记违规 / 取消」按钮(HTML mode);没配 → 走老「通过/拒绝」按钮路径。

2026-05-18 客户反馈:删除消息让**审查员处理**,不打扰商务/负责人。商务 / 负责人不参与删除消息合规审查,@ 他们只是噪音。

## 决策

`bot.py` 3 处改:删除预警的 mention 目标从 `owner_tg_id` 切到 `inspector_tg_id`(v3.0.15 加的字段)。其他链路不动:

| 预警类型 | @ 谁 | 字段 |
|---|---|---|
| 关键词预警 | 商务 | `business_tg_id` |
| 未回复 stage1(30 分钟) | 商务 | `business_tg_id` |
| 未回复 stage2(40 分钟) | 负责人 | `owner_tg_id` |
| **删除消息预警** | **审查员** | **`inspector_tg_id`** ← 本版改 |
| 账号失效(session revoked) | 审查员 | `inspector_tg_id`(v3.0.17 已是) |

### 具体改动

1. **`bot.py:262` `_route_callback` expected_actor 反查**(中央台路由 callback):
   ```python
   if atype == "deleted":
       expected_actor = account["inspector_tg_id"] or ""
   elif atype == "no_reply" and astage == 2:
       expected_actor = account["owner_tg_id"] or ""
   else:
       expected_actor = account["business_tg_id"] or ""
   ```
   (audit_log 写 expected_actor 用于怠工识别)

2. **`bot.py:343` `on_stage2_action` 本地 fallback expected_actor**(本地 bot 推送链路同样按 alert.type 分支)

3. **`bot.py:895` `send_delete_alert`**:
   ```python
   inspector_tg = account["inspector_tg_id"] if "inspector_tg_id" in account.keys() else ""
   owner_mention = (
       await self._build_tg_mention(inspector_tg, fallback_name="审查员") if inspector_tg else ""
   )
   ```
   变量名 `owner_mention` 沿用(只是装的是 inspector 的 mention,模板 `delete_alert(owner_mention=...)` 渲染不变),减少模板/调用方改动面。

4. **`templates/index.html:1047` 监察员字段 label 文案**:
   ```
   监察员 TG ID / 用户名 (账号失效时预警群 @ 此人)
   ↓
   监察员 TG ID / 用户名 (账号失效 + 删除消息预警 @ 此人,负责审核「登记违规 / 取消」)
   ```

### 向后兼容

- 账号**没配** `inspector_tg_id` → 走 v3.0.5 老「通过/拒绝」按钮路径(`owner_mention` 空 → 模板 fallback 老格式)
- 账号**配了** → @ 审查员 + 「登记违规/取消」按钮
- `inspector_tg_id` 字段已在 v3.0.15 migration V7 加,老客户升 v3.1.6 都有这字段(`account.keys()` 检查守住)

## 原因

1. **业务角色对齐**:删除消息合规审查由**审查员**负责,不是商务/负责人。@ 错人没人处理 + 商务收无用 ping。
2. **owner_tg_id 字段保留语义**:仍给 stage2 未回复预警用(负责人催商务 follow up),不混乱。
3. **inspector_tg_id 字段已存在**(v3.0.15 加,v3.0.17 用于账号吊销 @),复用不需要新加 DB 字段。
4. **变量名 owner_mention 沿用**:模板 `delete_alert(owner_mention=...)` 不动,只是 caller 传不同字段值。
