# ADR-0050: v3.1.7 — 预警文案/Sheet B3 改「中心/部门」格式 + 修 Sheet→DB 反向覆盖

**日期**:2026-05-18
**版本**:v3.1.7
**来源**:客户截图反馈 — 预警文案「中心/部门:恒睿公司-渠道中心」字段标签跟值顺序对不上,Sheet B3 同样反;web 后台改归属后 Sheet→DB 反向同步会把改的覆盖回去

## 背景

v3.0.15 引入「公司+中心」双下拉,前端拼接成 `公司-中心` 存 `account.company`(例:`鼎丰公司-商务中心`)。但:

1. **预警文案标签是「中心/部门」,值是「公司-中心」**:用户看到「中心/部门:恒睿公司-渠道中心」觉得反了,期望「渠道中心-恒睿公司」(中心在前)
2. **Sheet B3 同样反**:同步过去显示「公司-中心」格式,客户期望「中心/部门」格式
3. **web 改归属被反向覆盖**:`_sync_one_account_headers`(Sheet→DB 老路径)即使 DB 有值也会用 Sheet 当前值覆盖。v3.0.15 引入 DB→Sheet 单向覆盖后,两条路径打架:
   - DB→Sheet 把 `万有引力-渠道中心` 转格式写 Sheet B3 `渠道中心/万有引力`
   - Sheet→DB 读 B3 `渠道中心/万有引力` ≠ DB `万有引力-渠道中心` → 反向覆盖 DB
   - 结果 web 改的归属在下一轮 patrol 被 Sheet 老值压回去 / 或者格式被改

v3.1.3.5 ADR-0047 修过「Sheet 空 + DB 有值跳过」(防清空),但「DB 有值 + Sheet 不同」时仍会反向覆盖,本版补完最后一块。

## 决策

**1. 预警文案 + Sheet B3 写入时统一转「中心/部门」格式**

`templates.py` 加 `_swap_company(c)` helper,把 `account.company` 从 `公司-中心` 反转成 `中心-公司`。6 处文案模板(no_reply / no_reply_stage1 / no_reply_stage2 / keyword / deleted / session_revoked 等)统一用 `_swap_company(company)` 包一层。

`tasks.py _sync_account_business_to_sheet` 写 B3 前把 `db_company` 从 `公司-中心` 转 `中心/部门`(分隔符改成 `/` 对齐 sheet 上人填的格式)。

**2. Sheet→DB 老路径改成「DB 完全空才回写」**

`sheets.py _sync_one_account_headers` 之前逻辑:
```python
new_co = company if company else db_co
if new_co != db_co: 写 DB
```

改成:
```python
new_co = db_co if db_co else company  # DB 有值优先
if (new_op != db_op or new_co != db_co) and (not db_op or not db_co):  # 只在 DB 空才回写
    写 DB
```

= **DB 一旦有值,Sheet→DB 永远不动 DB**,web 优先 100% 落地。Sheet→DB 只兼容老 dept 首次接入(DB 完全空,从 Sheet 反向读出客户手填的初始值)。

**3. 数据迁移**

- 客户 web 改归属后,DB 存的还是 `公司-中心` 格式(不变)
- Sheet B3 显示的会从 `公司-中心` 变成 `中心/部门`(下一轮 patrol 自动迁移)
- 老 sheet 上手填的 `中心/部门` 格式(B3 原本就是)→ 跟新格式一致,不冲突

## 原因

- ✅ 文案跟字段标签一致,符合用户阅读习惯
- ✅ Sheet B3 跟客户手填格式一致(老 sheet 一直是「中心/部门」格式,人填的)
- ✅ web 优先 100% 落地,改归属不会被 Sheet 反向覆盖
- ✅ 0 数据迁移、0 schema 变、0 客户操作
- ✅ 老 dept 首次接入路径保留(Sheet→DB 只在 DB 空时回写)

## 来源

- 客户截图:截图 1「中心/部门:恒睿公司-渠道中心」字段顺序对不上
- 客户反馈:web 改归属后保存,下一轮 patrol 改的被覆盖回去
- demo 实测:bug 复现 + 修法验证

## 后果

- 中央台 `find_alert_route_by_company_field` 已支持 `-` / `/` 两种分隔符 + 公司-中心 / 中心-公司两种顺序,**不受影响**
- bot 推 `push_alert` payload `company` 字段仍是 `account.company`(`公司-中心` 格式),中央台正常路由
- 老 dept 首次接入(DB 空 + Sheet 手填):Sheet→DB 仍回写一次,之后 web 优先(行为不变)

## 文件清单

- `templates.py`:`_swap_company` helper + 6 处模板替换
- `tasks.py`:`_sync_account_business_to_sheet` 写 B3 前转格式
- `sheets.py`:`_sync_one_account_headers` 改成「DB 空才回写」
